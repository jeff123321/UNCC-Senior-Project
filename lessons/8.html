Lesson 8: Functions

Functions are a useful way to clean up your code and reduce how much typing you have to do. Functions go before main, but after all the #include and using namespace std statements.

Here is an example of a function.

The function is named sayHello. It takes a string for a name, and then it prints hello to the screen along with the name. In my main, I create a string name, and then I pass the name to the function, and the function does what it does with my name.

Let's look at the function more carefully.

void sayHello(string name)
{
      cout << "Hello " << name << "!";
}

void means that the function does not return any values. sayHello is the name of the function. In parenthesis are the arguments it will take. Inside the curly braces is the function body, which is what gets executed when the function is called.

Let's look at another example:

This function will give you a random number between 1 and 6, just like a die would. Instead of having to write out rand() % 6 + 1 over and over again, you can just call your rollDice() function.

Let's look at the function more closely.

int rollDice()
{
      //generates a random number between 1 and 6
      return rand() % 6 + 1;
}

This function returns an integer, that's what the int up front means. rollDice is the name of the function. In parentheses are the arguments that the function will take. This function takes no arguments, so there is nothing in the parentheses.

So let's summarize the process of writing a function:

1. First put the data type of the result of the function. This can be an integer, a boolean, a string, a double, a character, or anything you want. If you don't want to return anything, you just put void.

2. After the data type, you will put the name of the function.

3. In parentheses, put the arguments that your function will take. If there are no arguments you want to take, just leave the parentheses empty.

4. Then write the code for the function in curly braces {}.

Scope

Remember, when we were talking about scope in the section on loops. A variable that you create in your main will not be accessible to your functions. If you want a variable to be accessible to your functions, you will have to pass it as an argument, or you can do something else, which I will show you now in this example:

Look below the using namespace std line, you can see that I created a constant variable called SALES_TAX_RATE. Because I created this variable outside of all the functions, the variable is accessible by all of the functions. This is called a globally defined variable because you can access it from anywhere.

In general, it is not a good idea to use too many global variables. If you have a bunch of functions in your code, it can be quite challenging figuring out what exactly is happening to your variables. Basically, things can get out of hand quickly. However, making a constant a global variable is perfectly acceptable since nothing can change it anyway. SALES_TAX_RATE is a constant, so I made it global.

Call by reference

What if we want a function to change the value of the variables we pass to it? Here is an example of a program that does just that:

What the function does is takes in a number, and then cubes it, changing the value of the variable you passed to it. Look at the function again:

void cube(int& num)
{
      num = num * num * num;
}

The function is void because it isn't returning anything. Instead of having int num as an argument, it says int& num. This means that you are passing to the function the address of the variable, so that the function can actually make changes to it. Before, we had just been sending copies of our variables to our functions.

Passing arrays to functions

We can also pass arrays to functions. Here's how:

The program creates an array with 10 elements. Then it calls a function to print the array. Then it calls another function to clear out the array and make everything 0. Then it prints the array again.

This should help you to understand the utility of functions. We just eliminated a lot of repetitive code, and really cleaned up our main.

Let's look at the heading for the clearArray function:

void clearOut(int arr[], int length)

To pass the array, all we had to do was put square brackets after the name of the array. Note the name does not have to be the same as the array you passed to it. To make things easier, I also decided to take the length of the array of another argument. Other programming languages provide easy ways for you to get the length of an array, but not C++.

Also note that anytime you pass an array to a function, you are passing the array by reference, meaning that the array will not be copied over to the function. Instead you will be passing the address of the array, which means that you can change the contents of the array with your function. The reason for this is that for very large arrays, copying the contents would take a very long time.

How to put functions after main

I mentioned earlier that main itself is a function. It returns an integer, and it takes no arguments. However, main is special since it is where the main execution of a program will take place. Ordinarily, main has to go at the end, after all of the functions you have written.

However, you may want to have your functions go after main. The process of doing is kind of annoying. But here's how you do it:

The sayHello function is below the main, as we wanted. But look at the top here. You will see a line that looks like this:

void sayHello(string);

This is called a function prototype. It looks a lot like the function header for sayHello, but there are some differences. You do not need to name the variables that the function takes as arguments in your function prototype. Also, there is a semicolon at the end.

For further examples, here all function prototypes for all the functions we have done in this section:

int rollDice();
float totalCost(float);
void cube(int&);
void clearOut(int arr[], int);
void printArray(int arr[], int);

There are some things to note here. If you are passing a variable by reference, you will need to put an ampersand in your function prototype. If you are passing an array, you will need to name the array and then put the square brackets after the name of the array.
