If you don't want to download anything, you can go to repl.it which has an online IDE for C++. You can also use c9.io, although they will ask for your credit card.

Banking app

Create an app for a virtual bank. The app will first show you a menu like so:

  1. Sign in
  2. Create Account
  3. Exit

If the user enters in a number other than the ones that are on the menu, you can either show the menu again, or you can use the alarm character '\a' and say try again.

When the user has logged in, they will see another menu like so:

  1. Withdraw
  2. Make a deposit
  3. View balance
  4. Sign out

When the user chooses withdraw they will be asked to enter in how much money they want to take out, and the amount will be subtracted from their balance. Then they will be sent back to the previous menu. The rest of the options should be self explanatory.

Remember, use functions whenever you can like viewBalance() or withdraw(float amount) or something. Don't try to do everything in main.

For added challenge, see if you can make the information persist even after the program has exited. To do this, you will need to write the information to a text file. You can read about how to that here.

Virtual Casino

Create the casino program mentioned in the lessons. The program will have a menu like this:

  1. Baccarat
  2. Blackjack
  3. Craps
  4. Roulettes
  5. Slots
  6. View balance
  7. Quit

If you don't know the rules for these games you can look them up on the Internet. To make blackjack easier on yourself you can just make aces equal to 11 always, don't worry about suit, unless you really want to. Just generate a random number between 1 and 11 for the card values. For baccarat you can use the modulo operator to get the value of the cards.

We talked about random numbers in lesson 7. Also feel free to change the rules of the games to make things easier on yourself.

Remember to use functions!

Text based adventure game

This one is the most open ended. For an example of what I mean check out Dungeonman and also the sequel Dungeonman 2.

This one will involve a lot of typing, but it might be the most fun. Let the player use commands like go north, get/give item, or look to get an idea of their surroundings.

You might want to use a 2-d array to keep track of the map. Also, use function calls to get descriptions of the area you are in.

You don't have to get too nuts with this but do try and have fun!
